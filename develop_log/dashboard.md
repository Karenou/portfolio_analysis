# Streamlit 可视化看板设计文档

## 一、技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 框架 | Streamlit | 纯 Python，和现有代码无缝集成 |
| 图表库 | Plotly | Streamlit 原生支持，交互性好 |
| 热力图 | Plotly heatmap | 支持 hover 显示数值 |
| 表格 | st.dataframe | 内置排序/搜索，无额外代码 |
| KPI 卡片 | st.metric | 自带"涨跌"箭头指示 |
| 主题 | plotly_dark | 深色友好、简洁配色 |

## 二、文件结构

```
portfolio_analysis/
└── dashboard/
    ├── app.py              ← Streamlit 入口（Tab 路由）
    ├── pages/
    │   ├── overview.py     ← Tab1 概览
    │   ├── holdings.py     ← Tab2 持仓结构
    │   ├── portfolio.py    ← Tab3 组合分析
    │   └── asset_metrics.py← Tab4 个股体检
    └── utils/
        ├── data_loader.py  ← 统一数据加载层（从 SQLite + JSON 读取）
        └── chart_helper.py ← 图表生成辅助函数
```

## 三、看板整体架构

```
+-------------------------------------------------------------------+
|  Portfolio Dashboard                        最近更新：2026-05-22     |
+-------------------------------------------------------------------+
|                                                                     |
|  [概览]    [持仓结构]    [组合分析]    [个股体检]                       |
|                                                                     |
|  ================================================================  |
|                                                                     |
|                      当前 Tab 内容区                                  |
|            （长滚动，每个图表区块自带标题+说明+控件）                     |
|                                                                     |
+-------------------------------------------------------------------+
```

### Tab 逻辑

```
概览          ->  "我有多少、分在哪里"      （回答 WHAT）
持仓结构      ->  "具体持有什么、占比多少"    （回答 WHERE）
组合分析      ->  "整体表现 + 分散化评价"    （回答 HOW WELL）
个股体检      ->  "每个资产好不好、排名如何"   （回答 WHO）
```

从宏观到微观，逻辑递进。无全局侧边栏，控件内嵌在每个图表区块内。

### Tab 2：持仓结构 —— "我持有什么，仓位是否合理"

设计理念：从宏观到微观，Treemap 总览 -> 表格明细

- 面积 = 市值占比
- 颜色 = 资产大类（equity=绿, bond=蓝, commodity=橙）
- 图表上方说明文字："面积代表市值占比，颜色代表资产大类"
- 数据来源：classified_holdings

#### 区块 2：持仓明细（可排序表格）

- 内嵌控件：[筛选类别 ▾] [按市值排序 ▾]
- 列：名称、代码、平台、市值、占比、类别
- 数据来源：classified_holdings

#### 区块 3：一级分类分布（水平条形图）

- ETF / 个股 / 场外基金 / 货币基金 / 债基 占比
- 数据来源：各平台 penetration_summary -> level1_summary

### Tab 3：组合分析 —— "整体表现 + 分散化程度"

设计理念：先看组合整体表现，再看资产间关系（相关性合并于此）

#### 区块 1：组合表现 KPI

- 内嵌控件：[近1月] [近1年] 切换
- 三个指标卡：加权收益、加权波动率、组合夏普
- 数据来源：indicator_results -> portfolio_1m / portfolio_1y

#### 区块 2：资产大类相关性（热力图 + 文字评价）

- 左侧：Heatmap（equity / bond / commodity 两两相关系数矩阵）
- 右侧：文字评价
  - equity-bond 相关性 + 评级（低/中/高）
  - equity-commod 相关性 + 评级
  - bond-commod 相关性 + 评级
  - 总体分散化程度评价
- 图表上方说明："相关性越低，分散化越好；负相关资产能对冲风险"
- 数据来源：indicator_results -> correlation

#### 区块 3：风险收益全景（散点图）

- X轴 = 波动率（风险）
- Y轴 = 年化收益（回报）
- 气泡大小 = 市值占比
- 颜色 = 资产大类
- 虚线 = 无风险利率水平
- 图表上方说明："横轴=波动率（风险），纵轴=年化收益（回报），理想位置=左上角"
- 数据来源：indicator_results -> per_asset

### Tab 4：个股体检 —— "逐一审视每个资产"

设计理念：选中一个资产后展开详情，像体检报告一样

#### 区块 1：资产选择器

- 内嵌控件：[选择资产 ▾] [时间窗口：1M/1Y ▾]

#### 区块 2：选中资产 KPI（一行四个）

- 年化收益、波动率、最大回撤、夏普比率
- 数据来源：indicator_results -> per_asset

#### 区块 3：净值走势（归一化折线图）

- 选中资产的净值曲线（归一化到起点=1）
- 数据来源：fund_daily_nav / etf_daily_hist

#### 区块 4：排名信息

- 文字说明："在 N 个持仓中，夏普排第 X 名，回撤控制排第 Y 名"
- 数据来源：indicator_results 排序计算

#### 区块 5：全资产对比表（可排序表格）

- 内嵌控件：[按夏普排序 ▾] [筛选类别 ▾]
- 列：排名、名称、年化收益、波动率、最大回撤、夏普、类别
- 当前选中资产高亮显示
- 数据来源：indicator_results -> per_asset

## 五、设计原则

| 设计原则 | 来源 | 应用方式 |
|---------|------|---------|
| 每个图表自带"人话"解释 | Big Picture | 图表上方加一行说明 |
| 控件内嵌在图表区块 | Big Picture | 不用全局侧边栏，每个图自带切换按钮 |
| 长滚动布局 | Big Picture | 每个 Tab 内纵向滚动，不堆满一屏 |
| KPI 指标卡 | 黄金看板 | 顶部用数字卡片快速传递关键信息 |
| 框架思维组织 | 黄金看板 | Tab 划分有逻辑：全景->结构->组合->个体 |
| 深色友好 / 简洁配色 | 两者 | Plotly 的 plotly_dark 主题 + 统一色板 |
